# 스토리보드: 디자이너를 위한 바이브 코딩

> 씬 ID는 `02_스크립트/스크립트.md` 및 `03_스토리보드/design.md` 와 1:1 매핑.
> 시각 토큰·컬러·타이포·이징은 `design.md` 참조.
> Hyperframes 컴포지션 파일: `06_편집/hyperframes/index.html` (단일 파일 통합)
> 버전: v01 (2026-05-16)

---

## 글로벌 설정

- **해상도**: 1920×1080 / 30fps
- **세이프존**: 상하좌우 5%
- **배경**: `--bg-deep` (#0A0A0F) + `--grad-mesh-bg` 씬 전체에 은은히
- **자막 위치**: 하단 중앙, Y=940px, 글래스모피즘 박스 (design.md §3 참조)
- **로고 워터마크**: 우상단, opacity 0.25, 상시
- **BGM**: -22 LUFS, 발화 구간 -28까지 덕킹
- **씬 전환**: 0.4초 cross-fade (검은 프레임 노출 금지)

### 글로벌 GSAP timeline 구조

- 단일 `index.html` 안에 11개 `.scene` 컨테이너
- 각 씬: `tl.set("#scene-sNN", { opacity: 1 }, sceneIn)` / `tl.set("#scene-sNN", { opacity: 0 }, sceneOut)`
- 씬 내부 모션은 글로벌 시간 기준 GSAP `tl.fromTo()` + `tl.to()`
- **모든 주요 모션 라인 끝에 `// @sync:event_id`** 마킹 (design.md §5 표와 일치)
- 자막은 `tl.set(sub, { innerText: "..." }, t)` 패턴, sync_patcher 패치 대상 아님 (별도 동기화)

## 부록 C: 워크플로우 (자동 검증 루프 첫 적용)

이 영상은 새로운 **자동 검증 루프**를 처음 적용한다. 편집 단계 진입 후 다음 순서로:

1. **녹음 → mp4 수신** (`04_녹음/원본/2026-XX-XX_takeNN.mp4`)
2. **ffmpeg 분할 + Whisper 받아쓰기** (`04_녹음/가공/SNN.wav`, `04_녹음/받아쓰기/auto_transcript.srt`)
3. **Hyperframes index.html 작성** — 위 GSAP 코드의 시점들은 **추정값**. 모든 핵심 라인에 `// @sync:event_id` 마킹 (design.md §5 룰과 일치)
4. **sync_patcher 실행**:
   ```bash
   python3 도구/sync_patcher/sync_patcher.py \
     --transcript .../auto_transcript.srt \
     --design     .../design.md \
     --gsap       .../index.html \
     --sync-map   .../06_편집/검증/sync_map.json \
     --dry-run    # 먼저 미리보기
   ```
5. **Playwright DOM 검증**:
   ```bash
   node 도구/verify-playwright/verify.mjs \
     --sync-map .../sync_map.json \
     --html     .../index.html \
     --report   .../06_편집/검증
   ```
6. **Vision 시각 검증** (`도구/verify_vision.md` 절차)
7. 모두 PASS 시 `npx hyperframes render`
