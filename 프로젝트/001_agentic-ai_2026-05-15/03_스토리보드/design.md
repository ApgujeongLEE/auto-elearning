## 4. 발화-모션 싱크 룰 (sync_patcher 입력)

이 표는 `도구/sync_patcher`가 직접 읽어서 `sync_map.json`을 만들고 GSAP 코드를 패치합니다.

**규칙**:
- `trigger_word`: 받아쓰기에서 검색할 핵심 단어 (정확히 일치해야 함)
- `match_strategy`: `first` (첫 등장) / `nth=N` (N번째 등장) / `scene` (해당 씬 범위 안 첫 등장)
- `offset`: 발화 시작 시점에 더할 보정값(초). 양수면 발화 후, 음수면 발화 전
- `event_id`: GSAP 코드에서 패치할 식별자 (코드에 `// @sync:event_id` 주석으로 마킹)

| 씬 | event_id | trigger_word | match_strategy | offset | 의도 |
|----|----------|--------------|----------------|--------|------|
| S01 | `s01-check-in` | 알아서 | scene | +0.0 | "✓" 등장이 "알아서" 발화 순간과 동시 |
| S01 | `s01-subcopy-in` | 8분 | scene | -0.1 | 서브카피가 "8분만 보시면" 살짝 전에 |
| S02 | `s02-title-in` | 에이전틱 | scene | +0.0 | 타이틀이 "에이전틱 AI" 발화와 동시 |
| S03 | `s03-card-1-on` | 계획 | scene | +0.0 | 카드 1 활성화 = "계획" 발화 |
| S03 | `s03-card-2-on` | 도구 | scene | +0.0 | 카드 2 활성화 = "도구" 발화 |
| S03 | `s03-card-3-on` | 기억 | scene | +0.0 | 카드 3 활성화 = "기억" 발화 |
| S03 | `s03-card-4-on` | 점검 | scene | +0.0 | 카드 4 활성화 = "점검" 발화 |
| ... | ... | ... | ... | ... | ... |

**GSAP 코드 마킹 예시**:
```js
// 패치 전:
tl.fromTo("#s01-check", {...}, 3.8);   // @sync:s01-check-in

// sync_patcher가 자동으로 3.8 → 받아쓰기에서 "알아서" 시점 + offset 으로 교체
```

## 5. 자막 스타일 가이드

- 위치: 하단 중앙, 베이스라인 Y=920px
- 박스: `rgba(0,0,0,0.6)` 배경, 좌우 24px 상하 12px 패딩, border-radius 6px
- 글자: `#FFFFFF` Pretendard Medium 44pt, line-height 1.3
- **한 줄 최대 22자**, 줄바꿈 시 최대 2줄
- 노출 시간: **최소 1.2초, 최대 6초**
- 자막 ↔ 발화 끝나는 시점 정렬은 받아쓰기 word-level 시점에 자동 정렬

---

## 6. 모션 그래픽 원칙

- **들어옴/나감 페어**: 등장한 요소는 반드시 명시적으로 퇴장 (`tl.from` 단독 금지, `tl.fromTo` 또는 `from + to(opacity:0)` 쌍)
- **동시 모션 최대 2개**: 한 시점에 움직이는 요소 2개 초과 금지
- **강조 모션은 회당 1회**: 같은 요소에 바운스/스케일을 0.5초 안에 두 번 적용 금지
- **트랜지션 0.3초 이상**: 씬 전환은 페이드/와이프 0.3초 이상 (시청자 시선 따라잡기)
- **씬 사이 흑백 프레임 금지**: 한 씬 페이드아웃과 다음 씬 페이드인이 겹치도록 (`tl.set(scene-1, {opacity:0}, T)` 와 `tl.set(scene-2, {opacity:1}, T)` 같은 시점)

## 7. 자동 품질 체크리스트

`도구/verify_playwright`가 자동 검사하는 항목:

- [ ] 모든 `@sync:` 마킹된 모션이 발화 시점과 ±0.15초 이내
- [ ] 한 시점에 active(`opacity>0.1`)인 `.scene` 컨테이너가 정확히 1개
- [ ] 자막 한 줄이 22자 이하
- [ ] 자막 노출 시간 1.2~6.0초 사이
- [ ] 씬 전환 시 동시에 보이는 씬이 0~1개 (블랙 프레임 없음)
- [ ] 동시 모션 트윈 ≤ 2 (각 시점 0.1초 윈도우 내)

`도구/verify_vision`로 사람·Vision LLM이 검사하는 항목:

- [ ] 컬러 대비 4.5:1 이상 (자막 가독성)
- [ ] 발화 키워드와 화면 강조 키워드가 정확히 일치 (예: "가트너" 발화 시 화면에 "GARTNER")
- [ ] 의도된 무드(차분/임팩트 등) 살아 있음
- [ ] 정보 과밀 없음 (한 화면 텍스트 6라인 이하)
